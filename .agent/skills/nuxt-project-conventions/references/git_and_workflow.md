## Version Control & Commit Messages
When a feature is completed, you MUST generate a clean, easy-to-read commit message and/or Pull Request description using basic, simple English.

- **Title Format**: Provide a clear, descriptive title summarizing the overall feature (e.g., "Add attendance & score report export to Excel").
- **Description Format**: Use a bulleted list to detail the specific changes made in the feature branch. 
- **Action Verbs**: Start every bullet point with a simple, present-tense action verb (e.g., Add, Include, Update, Remove, Improve, Simplify, Refactor, Clean up).
- **Simplicity**: Avoid overly complex jargon. Write the descriptions so that a basic English user can easily understand exactly what was added, modified, or removed in the codebase.
